# Final pass · checklist

Run this on every draft before returning it. The list is short on
purpose: each item is binary, no judgment required.

## Em dashes

- [ ] Zero em dashes in the body. If one survives, it has a specific
      reason and a comment explaining why.

## Filler

- [ ] No "importantly", "crucially", "notably", "fundamentally"
- [ ] No "it's worth noting that"
- [ ] No "in some sense", "essentially", "basically"
- [ ] No "really", "very", "quite", "rather" used as intensifiers

## Rule-of-three

- [ ] Every list of three items has been read against this question:
      do these escalate? Does each carry distinct content?
- [ ] Lists that fail the test have been cut to one or two items.

## Balanced pairs

- [ ] No "X is not Y, it is Z" unless the contrast does real work.
- [ ] If a pair survives, it pulls its weight; the negation is not
      filler.

## Headers

- [ ] Headers are functional, not tagline-shaped.
- [ ] No header reads like it was meant to be screenshot.

## Sentence rhythm

- [ ] First sentence of each paragraph is short.
- [ ] No sentence runs more than two clauses without a break.
- [ ] Lengths vary; the draft does not read in monotone.

## Voice

- [ ] If a line could survive in a meeting transcript, it does.
- [ ] Active voice. Plain verbs.
- [ ] Concrete: numbers, names, examples, not abstractions.

## Read aloud

- [ ] You read it aloud once. Nothing caught in your mouth.

If any item fails, fix it. Do not return the draft until every box is
ticked.
